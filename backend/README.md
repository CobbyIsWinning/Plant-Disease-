FastAPI backend for Plant Disease Prediction

This folder contains a minimal FastAPI app that loads a trained MobileNetV2 model and exposes a /predict endpoint.

Files created:
- app.py            # FastAPI app
- requirements.txt  # runtime dependencies
- saved_models/     # place your saved model and class indices here

How to run locally

1. Create a virtual environment and install dependencies

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Place your trained model and class indices

Copy your model file (one of):
- mobilenetv2_best.keras
- mobilenetv2_final.keras

into `backend/saved_models/`.
Also copy the class indices JSON generated by the notebook as:
- `class_indices_mobilenetv2.json`

3. Start the API server

```bash
# from repository root or backend/ folder
uvicorn backend.app:app --reload --host 0.0.0.0 --port 8000
```

4. Test with curl

```bash
curl -F "file=@/path/to/leaf.jpg" "http://127.0.0.1:8000/predict"
```

Response example:

```json
{
  "predicted_class": "Tomato_healthy",
  "confidence": 0.92,
  "top_k": [
    {"class": "Tomato_healthy", "confidence": 0.92},
    {"class": "Tomato_Early_blight", "confidence": 0.05}
  ]
}
```

Notes
- The server expects a MobileNetV2-style preprocessing; the notebook's transfer-learning cell saves models compatible with this API.
- If the server reports the model is not loaded, ensure the model file and class indices are in `backend/saved_models/`.
